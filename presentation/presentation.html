Intro · What we built

GIX — the GPU Inference Exchange

A decentralized, hardware-verified spot market for GPU inference, settled on Sui. Compute is matched on an order book, every job is proven by attestation and verified on-chain, and payment settles autonomously — no trust in any single provider, and no black-box API.

Sui Move package · gix Provider node · TEE + Ollama OpenAI-compatible gateway TypeScript SDK Web trading console Simulation harness

A full vertical slice runs today — real inference, bought on-chain, attested, settled, hash-verified. Live on Sui testnet.

GPU INFERENCE EXCHANGESUI · DEEPBOOK · WALRUS
Problem statement

Compute is priced like real estate — illiquid, opaque, and unverifiable.

GPU Providers
  • Idle / spare capacity. Stranded GPU hours have no neutral, permissionless venue to be sold per-job.
  • Opaque pricing. List prices, not a real market — no continuous, neutral spot price for compute.
GPU Users
  • Locked capacity. You rent a whole GPU by the hour and pay for idle time; bursty AI workloads waste it.
  • Black-box trust & no audit trail. An API can silently swap models, quantize, or truncate — with no tamper-evident record binding model · input · output.
  • Expensive API costs. You overpay for committed capacity and premium per-token pricing.

The result: AI builders overpay for committed capacity and still can't prove they got what they paid for.

THE GAPILLIQUID · OPAQUE · UNVERIFIABLE
Purpose & vision

Make compute a liquid, verifiable commodity.

GIX turns GPU inference into something you can price on a real market, prove was executed correctly, and settle without a middleman — cheaper than renting your own GPU, and verifiable in a way no black-box API can match.

For builders
One API, pay per call, provable execution — ideal for agents and trust-sensitive AI.
For providers
Sell spare/attested capacity per-job on a neutral venue — revenue, not lock-in.
For the ecosystem
Neutral market infrastructure — a public spot price for compute, settled on Sui.

Verification & trust · the moat

What the contract checks, on-chain

  • Vendor signature chain → governance-pinned roots
  • Measurement allowlisted → the right model/runtime ran
  • Hashes bind the Job → model · input · output match
  • Within SLA · fresh nonce → on time, no replay

No trust in any operator — the contract verifies the proof. v1 uses hardware TEE attestation; the interface is pluggable so a zk backend can be added later.

How you use it · product surfaces

Plug in like any inference API — provably served

OpenAI-compatible Gateway
Point your existing client at one endpoint. The matching, escrow, attestation, and settlement are abstracted away — USDC in, verified tokens out.
Web trading console
Markets sidebar (GPU Compute ⇄ Crypto Pairs), order book, order ticket, wallet, the SUI→USDC on-ramp, and a hash-verifiable result viewer.
TypeScript SDK
Quote → order → await → fetch → independently verify, plus the provider flow. Trust-minimized: nothing is "verified" unless it checks from chain + Walrus.
Provider node + GPU-less client
A provider serves real inference (Ollama runtime) and signs attestations; a buyer on another machine purchases on-chain and verifies — buyer ≠ seller.

The wedge isn't cheapest tokens — it's verifiable, neutral, pay-per-use compute for agentic, regulated, bursty, and long-tail workloads.
